# Evaluation fixtures

`scores.csv` is the **frozen evaluation set** the feotest contracts certify
against. Each row is one scored histopathology patch:

| column         | meaning                                                        |
|----------------|----------------------------------------------------------------|
| `patch_id`     | stable identifier within the evaluation split                  |
| `label`        | ground truth: `tumour` or `normal`                             |
| `score`        | model's predicted probability the patch shows tumour, `[0,1]`  |
| `inference_ms` | wall-clock latency of that single inference, milliseconds      |

Lines beginning with `#` are provenance comments. The decision threshold that
turns a `score` into a tumour/normal call is applied by the contract, not
baked into the fixture — so the same scores can be re-certified at a different
operating point.

## Reproducibility contract — scores committed, images fetched by script

Only the **scores** live here. The raw histopathology images are **never
committed**: `scripts/regenerate_scores.py` fetches the PatchCamelyon (PCam)
evaluation split via `torchvision` into a git-ignored `data/` directory, runs
your evaluated model over it, and records one row per patch. This keeps the
repo small and sidesteps any dataset-redistribution question while remaining
fully reproducible: anyone with the model can regenerate byte-comparable
scores.

```bash
# Real mode — your evaluated TorchScript model over the PCam test split:
python scripts/regenerate_scores.py --model path/to/model.pt --limit 2000

# Model-free stand-in (what is committed here):
python scripts/regenerate_scores.py --placeholder
```

## ⚠ The committed `scores.csv` is a PLACEHOLDER

The version checked in is **synthetic, deterministic stand-in data** (seed
`20260605`), generated by `--placeholder` so the showcase runs on a fresh
clone with no model or dataset download. It is **not** real model output and
must not be cited as a performance result. Replace it by running real mode
with a genuine tumour-detection model before drawing any conclusion. The
placeholder marker is carried in the file's own header comments.
